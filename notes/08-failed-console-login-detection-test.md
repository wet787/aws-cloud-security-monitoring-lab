# 🔐 Failed AWS Console Login Detection Test

## 🎯 Objective

The objective of this step was to test the Failed AWS Console Login detection by generating a controlled failed sign-in event against the AWS Management Console.

## 🧪 Test Setup

The Failed Console Login detection was created using a CloudWatch metric filter that searches CloudTrail logs for `ConsoleLogin` events with a login result of `Failure`.

A controlled failed login attempt was generated by attempting to sign in to the AWS Management Console with the correct IAM username and an intentionally incorrect password.

## ✅ Actions Completed

- Created a CloudWatch metric filter named `FailedConsoleLogins`

- Created a CloudWatch alarm named `FailedConsoleLoginAlarm`

- Connected the alarm to the existing SNS topic `aws-security-alerts`

- Opened an incognito/private browser window

- Attempted to sign in to the AWS Management Console with an incorrect password

- Generated a failed console login event

- Reviewed the CloudWatch alarm graph

- Confirmed that the `FailedConsoleLoginCount` metric showed activity

- Confirmed that the existing `UnauthorizedAPICallsAlarm` also triggered during testing

## 🔁 Detection Flow

```text

Incorrect AWS Console password entered

&#x20;       ↓

AWS rejected the login attempt

&#x20;       ↓

CloudTrail recorded a ConsoleLogin failure event

&#x20;       ↓

CloudWatch metric filter matched the event

&#x20;       ↓

FailedConsoleLoginCount metric showed activity

&#x20;       ↓

CloudWatch alarm evaluated the failed login metric

&#x20;       ↓

SNS notification channel was available for alerting

```

## 📊 Result

The failed console login test successfully generated metric activity for the `FailedConsoleLoginAlarm`. The alarm graph showed a spike in the `FailedConsoleLoginCount` metric after the failed login attempt was generated.

The alarm later returned to an OK state after the evaluation period ended, which is expected behavior when no additional failed login attempts occur.

During testing, the existing `UnauthorizedAPICallsAlarm` also entered the alarm state. This showed that failed authentication activity may generate additional unauthorized or denied activity in CloudTrail and should be reviewed during investigation.

## 🧠 SOC Analyst Notes

This test validates that failed AWS Console login attempts can be detected using CloudTrail events delivered to CloudWatch Logs. Failed login attempts may occur because of normal user error, but repeated failures, unexpected source IP addresses, or attempts against privileged users may indicate credential guessing or attempted account compromise.

This test also demonstrates why analysts should review related alarms together. In this lab, the Failed Console Login detection showed activity, while the existing Unauthorized API Calls alarm also triggered. Reviewing both alerts together provides better context during investigation.
